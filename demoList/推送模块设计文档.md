# 推送功能模块设计文档

## 1. 项目概述

本项目实现一个可配置的多平台推送管理模块，支持对接不同平台的API进行推送，并提供配置化、历史记录和群组管理功能。

## 2. 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Vue 3 + Element UI Plus + JavaScript |
| 后端 | Java 21 + Spring Boot 3.4.4 |
| 数据库 | MySQL 8.0+ |

## 3. 架构设计原则

- **简洁性**：避免过度设计，用最简单的方案满足需求
- **扩展性**：通过抽象接口支持新增推送平台
- **配置化**：核心参数通过配置管理，减少硬编码
- **可追溯**：所有推送操作留痕

## 4. 数据库设计

### 4.1 推送平台表 (`push_platform`)
存储支持的推送平台信息

```sql
CREATE TABLE push_platform (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    platform_code VARCHAR(50) NOT NULL UNIQUE COMMENT '平台编码：W3_TODO, WELINK_APP',
    platform_name VARCHAR(100) NOT NULL COMMENT '平台名称',
    api_endpoint VARCHAR(255) COMMENT 'API地址（可选，配置中覆盖）',
    enabled TINYINT DEFAULT 1 COMMENT '是否启用 0-禁用 1-启用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推送平台表';
```

### 4.2 业务类型表 (`business_type`)
存储业务类型信息

```sql
CREATE TABLE business_type (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    business_code VARCHAR(50) NOT NULL UNIQUE COMMENT '业务编码：QA_REPORT, TEST_REPORT',
    business_name VARCHAR(100) NOT NULL COMMENT '业务名称',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='业务类型表';
```

### 4.3 推送配置表 (`push_config`)
业务类型与推送平台的配置关系

```sql
CREATE TABLE push_config (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    business_type_id BIGINT NOT NULL COMMENT '业务类型ID',
    platform_code VARCHAR(50) NOT NULL COMMENT '平台编码',
    enabled TINYINT DEFAULT 1 COMMENT '是否启用',
    config_json TEXT COMMENT '推送配置JSON（静态参数）',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    UNIQUE KEY uk_business_platform (business_type_id, platform_code),
    FOREIGN KEY (business_type_id) REFERENCES business_type(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推送配置表';
```

### 4.4 推送历史表 (`push_history`)
所有推送记录

```sql
CREATE TABLE push_history (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    business_type_id BIGINT NOT NULL COMMENT '业务类型ID',
    platform_code VARCHAR(50) NOT NULL COMMENT '平台编码',
    group_id BIGINT COMMENT '使用的群组ID',
    business_key VARCHAR(100) COMMENT '业务主键（如报告ID）',
    request_json TEXT NOT NULL COMMENT '完整请求参数',
    response_json TEXT COMMENT '响应结果',
    status TINYINT DEFAULT 0 COMMENT '状态 0-失败 1-成功',
    error_message TEXT COMMENT '错误信息',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    INDEX idx_business_type (business_type_id),
    INDEX idx_platform (platform_code),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推送历史表';
```

### 4.5 推送群组表 (`push_group`)
推送群组信息

```sql
CREATE TABLE push_group (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    group_name VARCHAR(100) NOT NULL COMMENT '群组名称',

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'

```

### 4.6 群组成员表 (`group_member`)
群组成员关系

```sql
CREATE TABLE group_member (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    group_id BIGINT NOT NULL COMMENT '群组ID',
    employee_no VARCHAR(50) NOT NULL COMMENT '工号',
    employee_name VARCHAR(100) COMMENT '姓名',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    UNIQUE KEY uk_group_employee (group_id, employee_no),
    FOREIGN KEY (group_id) REFERENCES push_group(id) ON DELETE CASCADE
```


### 6.2 核心组件设计

#### 6.2.1 PushDialog.vue - 推送预览&执行弹窗（公共组件）

这是**最重要的公共组件**，供业务系统调用来执行推送操作。

**功能特性：**
- 显示业务类型和已配置的推送平台
- 实时预览每个平台的完整推送参数
- 群组选择（切换群组自动刷新预览）
- 格式化展示参数值（支持数组、对象、字符串）
- 二次确认机制防止误操作
- 快捷入口：历史记录、群组配置、业务类型配置

**Props 参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `visible` | Boolean | 是 | 控制弹窗显示/隐藏（支持 v-model） |
| `businessCode` | String | 是 | 业务编码（如：QA_REPORT） |
| `dynamicParams` | Object | 是 | 动态参数（如：{title, content, jumpUrl}） |
| `businessKey` | String | 否 | 业务主键（用于记录关联，如报告ID） |

**Events 事件：**

| 事件名 | 参数 | 说明 |
|--------|------|------|
| `update:visible` | Boolean | 弹窗显示状态变化 |
| `success` | - | 推送成功后触发（可用于刷新列表等） |

### 6.3 API 封装说明

#### 6.3.1 Axios 拦截器（api/request.js）

```javascript
// 请求拦截器：添加 token 等认证信息（如需要）
// 响应拦截器：统一处理 ApiResponse 结构
axios.interceptors.response.use(
  response => {
    // 后端返回 { code: 200, message: "", data: {} }
    if (response.data.code === 200) {
      return response.data  // 直接返回 data
    }
    return Promise.reject(response.data)
  },
  error => {
    return Promise.reject(error)
  }
)
```


### 6.5 参数说明

#### 6.5.1 业务编码（businessCode）

用于标识不同的业务场景，需在数据库 `business_type` 表中预先配置：

| 业务编码 | 业务名称 | 说明 |
|----------|----------|------|
| QA_REPORT | QA报告 | QA相关报告推送 |
| TEST_REPORT | 测试报告 | 测试报告推送 |
| ... | ... | 按需扩展 |

#### 6.5.2 动态参数（dynamicParams）

推送时传递的动态参数，根据不同平台有不同结构：

**W3 待办平台：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `taskDesc` | String | 是 | 任务描述（建议加上标题、流程和状态） |
| `taskURL` | String | 是 | 待办详情跳转 URL（绝对地址） |
| `taskUUID` | String | 否 | 任务全局唯一标识（不传会自动生成） |
| `time` | String | 否 | 业务发送时间（不传使用当前时间） |
| `applicant` | String | 否 | 申请人（默认：system 00000000） |
| `reserve3` | String | 否 | 申请详情跳转 URL（默认使用 taskURL） |

调用示例：

```javascript
{
  taskDesc: 'QA周报 - 第42周 - 已完成，请查阅',
  taskURL: 'https://example.com/qa/report/123',
  taskUUID: 'qa_report_abc123',  // 可选
  // time, applicant 会自动生成
}
```

**WeLink 应用号平台：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` 或 `titleName` | String | 是 | 模板标题参数（映射到 tName） |
| `content` 或 `contentName` | String | 是 | 模板内容参数（映射到 cName） |
| `action` | String | 否 | 动作说明（映射到 doName） |
| `jumpUrl` | String | 否 | 跳转 URL |

调用示例：

```javascript
{
  title: 'QA周报',           // 会转换为 {"tName": "QA周报"}
  content: '第42周报告已生成', // 会转换为 {"cName": "第42周报告已生成"}
  action: '请查阅',          // 会转换为 {"doName": "请查阅"}
  jumpUrl: 'https://example.com/report/123'
}
```

**注意：**
- WeLink 的 `templateTitleParams` 和 `templateContentParams` 会自动转换为 JSON 字符串格式
- W3 的 `handler`（责任人）会自动从群组成员中提取并格式化为 `z 工号` 格式

#### 6.5.3 业务主键（businessKey）

用于关联业务数据，记录到历史表中方便追溯：

```javascript
businessKey: 'report-123'  // 或其他业务唯一标识
```

## 8. 推送平台参数结构

根据外部推送平台接口文档，各平台的参数结构如下：

### 8.1 W3 待办平台

**接口地址：** `http(s)://w3-beta.huawei.com/task/rest/todotaskmanagement/v2/createtask4w3`

**静态配置参数（config_json）：**

```json
{
  "appName": "QA报告系统",           // 应用名称（展示在 W3 首页）
  "appURL": "http://w3-beta.huawei.com",  // 应用待审批任务列表 URL
  "type": "J2EE",                   // 任务类型（固定值）
  "reserve1": "",                   // 必填字段（可为空）
  "reserve2": "qa_report",          // 备用字段，区分任务类型
  "reserve10": "0"                  // 是否支持移动审批（0-不支持）
}
```

**动态参数：** taskDesc、taskURL、taskUUID、time、applicant、reserve3（见第 6.5.2 节）

**完整请求参数示例：**

```json
{
  "appName": "QA报告系统",
  "appURL": "http://w3-beta.huawei.com",
  "type": "J2EE",
  "reserve1": "",
  "reserve2": "qa_report",
  "reserve10": "0",
  "taskDesc": "QA周报 - 第42周 - 已完成，请查阅",
  "taskURL": "https://example.com/qa/report/123",
  "taskUUID": "qareport_abc123",
  "time": "2025-01-22 16:30:00",
  "applicant": "system 00000000",
  "reserve3": "https://example.com/qa/report/123",
  "handler": "z c00807372,z c00807373"  // 从群组成员自动生成
}
```

**接口要求：**
- handler 格式：`z 工号`（多个用逗号分隔）
- taskUUID 必须全局唯一，不能有特殊字符
- time 格式：`yyyy-MM-dd HH:mm:ss`

### 8.2 WeLink 应用号平台

**接口地址：** `http://kweuat.huawei.com/feedmsg/publicservices/template/sendTemplateMessage`

**静态配置参数（config_json）：**

```json
{
  "app_id": "com.huawei.feed",           // HIS APPID
  "theme_id": "d99b0ccaad7a4395b5f2aa558218dc9c",  // 应用号 ID
  "templateNo": "20251223000522",         // 模板编号
  "from_user_account": "z00512371",       // 发送人账号
  "type": "1",                            // 事件类型（1-通知类）
  "displayType": "text",                  // 消息显示类型
  "noticeType": 1                         // 通知类型
}
```

**动态参数：** title、content、action、jumpUrl（见第 6.5.2 节）

**完整请求参数示例：**

```json
{
  "app_id": "com.huawei.feed",
  "theme_id": "d99b0ccaad7a4395b5f2aa558218dc9c",
  "templateNo": "20251223000522",
  "from_user_account": "z00512371",
  "type": "1",
  "displayType": "text",
  "noticeType": 1,
  "templateTitleParams": "{\"tName\":\"QA周报\"}",           // 自动转换
  "templateContentParams": "{\"cName\":\"第42周报告已生成\",\"doName\":\"请查阅\"}",  // 自动转换
  "to_user_account": "c00807372,c00807373",                  // 从群组成员自动生成
  "jump_url": "https://example.com/report/123"
}
```

**接口要求：**
- templateTitleParams 和 templateContentParams 必须是 JSON 字符串格式
- to_user_account 格式：工号1,工号2（用逗号分隔，最多 200 人）
