## welink应用号

接口路径 http://kweuat.huawei.com/feedmsg/publicservices/template/sendTemplateMessage

接口入参
{
"app_id": "com.huawei.feed",
"theme_id": "d99b0ccaad7a4395b5f2aa558218dc9c",
"type": "1",
"templateNo": "20251223000522",
"templateTitleParams": "{\"tName\":\"足球\"}",
"templateContentParams": "{\"cName\":\"足球比赛\",\"doName\":\"请大家提前做好准备\"}",
"from_user_account": "z00512371",
"to_user_account": "z00512371,dwx477491",
"jump_url": "http://feed.cloud.huawei.com/feedManagementWeb/#/home/appNumberList&welink_open_uri=aDU6Ly83ODE1ODA5MDI5MDI4MDQvaHRtbC9pbmRleC5odG1sP3VybD0=",
"displayType": "text",
"noticeType": 1
}

接口入参说明
属性名称 属性类型 是否必填 描述
app_id (100 Byte) 字符串 是 HIS APPID
theme_id (100 Byte) 字符串 是 应用号 ID
type (10 Byte) 字符串 否 事件类型 1 通知类
templateTitleParams(500 Byte) 字符串 是 消息标题参数化（500 字节）
templateContentParams (2000 Byte) 字符串 是 消息内容参数化（2000 字节），如果消息内容没有参数，templateContentParams对应参数值填“{}”
templateNo(100 Byte) 字符串 是 模板编号
from_user_account(200 Byte) 字符串 否 发送人（200 字节）
to_user_account (4000 Byte) 字符串 是 接收者 w3 账号（一次调用限 200 人，用符号,或，或:或：或;或；或、分隔，如:z00512371,dwx477491）
jump_url (1500 Byte) 字符串 否 跳转 URL（PC 端和 移动端可配置不同的 URL）查看拼接方法
displayType(100 Byte) 字符串 否 消息显示类型 为空:默认文本

Authorization：{动态token}

入参说明

| **属性名** | **数据类型**   | **说明**                                                                                                                                                                                                                                                                                                                                                               |
| ---------- | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| appName    | VARCHAR2(250)  | 应用名称，即在W3首页展示的列表名称。在w3首页展示的名称，唯一，区分大小写。中英文页面展示一致。不要有特殊字符例如:**​'/:&#^\*(?<>​**等。待办应用名规范要求一般为：领域名称*中文电子流名称中文名(英文名)，例如：iHR*人力资本(HCM)                                                                                                                                        |
| appURL     | VARCHAR2(1500) | 访问应用待审批任务列表的URL（w3后台用，前台不展示），地址请提供正确，不要传w3首页或者w3接口地址                                                                                                                                                                                                                                                                        |
| type       | VARCHAR2(100)  | 任务类型，统一为J2EE，以前有notes（已下线）。部分系统用来做区分待办类型                                                                                                                                                                                                                                                                                                |
| taskDesc   | VARCHAR2(2500) | 任务描述建议加上标题、流程和状态，不要单一（单一对用户体验不友好）                                                                                                                                                                                                                                                                                                     |
| taskURL    | VARCHAR2(1500) | W3首页“我的待办”详情跳转URL，绝对地址，相对地址可能导致访问404                                                                                                                                                                                                                                                                                                         |
| taskUUID   | VARCHAR2(300)  | 任务全局唯一标示符。建议不要用中文，**​不能重复使用，不建议带用户工号信息（华为账号会切换，可能导致删除失败），建议拼接上应用名英文简写。​**不要有特殊字符例如: **'/:&#^\*(?<> 空格**等（特殊字符会导致删除接口异常）                                                                                                                                                  |
| time       | VARCHAR2(50)   | 业务系统发送待办的时间，格式必须为：yyyy-mm-dd hh24:mi:ss                                                                                                                                                                                                                                                                                                              |

| reserve10  | VARCHAR2(300)  | 是否支持移动审批——在移动管理后台需要注册配置信息安全浏览器审批、原生审批：与移动端注册时reserve10匹配，相同则说明该待办支持移动端安全、原生审批不传值默认为赋值0WE码审批：固定521,**不支持WE码审批不要传521，否则在移动端查看待办会闪屏、或者移动待办总数不对。**​**这个字段只是w3筛选后在移动端显示，不代表待办一定能在移动审批，需要业务系统实现了移动待办功能才行** |
| reserve1   | VARCHAR2(300)  | rest接口必传，可以传""                                                                                                                                                                                                                                                                                                                                                 |
